# Contribution Guidelines

Vielen Dank für Ihr Interesse am Digital Signage Projekt!

## Wie kann ich beitragen?

### Bugs melden

- Verwenden Sie die GitHub Issues
- Beschreiben Sie das Problem detailliert
- Fügen Sie Screenshots hinzu
- Geben Sie Ihre Umgebung an (OS, Version, etc.)

### Features vorschlagen

- Öffnen Sie ein Issue mit dem Label "enhancement"
- Beschreiben Sie den Use Case
- Erklären Sie, warum das Feature nützlich wäre

### Code beitragen

1. Fork das Repository
2. Erstellen Sie einen Feature-Branch (`git checkout -b feature/AmazingFeature`)
3. Committen Sie Ihre Änderungen (`git commit -m 'Add some AmazingFeature'`)
4. Pushen Sie den Branch (`git push origin feature/AmazingFeature`)
5. Öffnen Sie einen Pull Request

## Code-Standards

### C# / .NET
- Befolgen Sie die Microsoft C# Coding Conventions
- Verwenden Sie async/await für I/O-Operationen
- Schreiben Sie XML-Dokumentation für öffentliche APIs
- Nutzen Sie nullable reference types

### Python
- Befolgen Sie PEP 8
- Verwenden Sie Type Hints
- Schreiben Sie Docstrings für alle Funktionen
- Nutzen Sie async/await für I/O

### Commit-Nachrichten

Verwenden Sie klare, beschreibende Commit-Nachrichten:

```
Add feature: Brief description

Detailed explanation of what was added and why.
```

Präfixe:
- `Add:` Neue Features
- `Fix:` Bugfixes
- `Update:` Änderungen an bestehenden Features
- `Refactor:` Code-Verbesserungen ohne Funktionsänderung
- `Docs:` Dokumentationsänderungen
- `Test:` Test-Ergänzungen

## Testing

- Fügen Sie Unit-Tests für neue Features hinzu
- Stellen Sie sicher, dass alle Tests erfolgreich sind
- Streben Sie >70% Code Coverage an

## Pull Request Process

1. Aktualisieren Sie die README.md mit Details zu Änderungen
2. Aktualisieren Sie die Versionsnummer (SemVer)
3. Der PR wird von mindestens einem Maintainer reviewed
4. Nach Approval wird der PR gemerged

## Verhaltenskodex

- Seien Sie respektvoll
- Konstruktive Kritik ist willkommen
- Keine Beleidigungen oder Diskriminierung
- Helfen Sie neuen Contributors

## Fragen?

Bei Fragen können Sie:
- Ein Issue öffnen
- Die Diskussionen nutzen
- Eine E-Mail an die Maintainer senden
